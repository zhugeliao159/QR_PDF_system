新增或调整内容：

- `bindings` 增加 `title`、`display_code`、`grade`、`subject`、`textbook_version`、`chapter`。
- `pdf_jobs` 增加 `qr_mode` 和 `qr_version_id`。
- 新增 `version_references`，记录固定二维码或固定 PDF job 对版本的保护引用。

## 认证与安全

- 管理员用户名由 `ADMIN_USERNAME` 配置，当前为 `admin`。
- 密码使用带随机盐的 scrypt 哈希，明文不写入 Git、README 或 `.env`。
- 首次密码和 `SESSION_SECRET` 由安全初始化脚本生成；`.env` 权限为仅所有者可读写。
- Session 使用签名 Cookie，设置 `HttpOnly`、`SameSite=Lax`、8 小时有效期；HTTPS 部署时可启用 `Secure`。
- 所有管理状态修改表单校验与 Session 绑定的 CSRF token。
- 登录有失败速率限制；会话过期返回登录页。
- Swagger/OpenAPI 默认关闭，普通管理员页面没有 API 文档入口。
- 容器继续以非 root 用户运行，`cap_drop: ALL`、`no-new-privileges`、日志轮转、512 MiB 内存和 128 PIDs 限制保持有效。

## 中文文件名验证

自动化测试使用 `高一数学第一章解析.pdf` 和乱码样本验证了 NFC 规范化、路径与控制字符清理、数据库显示、页面显示、下载文件名和 RFC 5987 `filename*`。

实际历史资料 `学术英语理工Unit 1 202409.pdf` 已能在工作台、资料列表、详情和 PDF 结果页正确显示。其文件内容 SHA-256 仍为 `bd4b3408f4986d4a41f5071c2d645328edf70882181438697be849331eb6f0fc`。

## 动态与固定版本验证

- 动态入口在替换和恢复后跟随当前版本。
- 固定入口在替换和恢复后仍返回指定版本。
- 不允许用一个资料的 `version_id` 访问另一个资料。
- 固定二维码和固定 PDF job 会写入版本引用。
- 自动清理保留当前版本、受保护版本及最多 5 个普通历史版本。

以上行为由自动化测试覆盖；实际旧动态二维码也重新下载并核对了 SHA-256。

## PDF 管理流程

- 管理员从可搜索的资料列表选择资料，不需要输入 `qr_id`。
- 支持第一页起算的页码、四角位置、小/标准/大尺寸、动态/固定方式及高级边距。
- 生成前校验文件、PDF 页数、页码和可放置区域。
- 结果页按需渲染目标页预览，可下载处理后的 PDF。

浏览器手工检查了工作台、资料列表、真实资料详情、PDF 向导和已有真实结果预览。预览中二维码位于目标页右下角。390x844 和 1440x900 视口均无页面级横向溢出。实际文件选择和提交由自动化集成测试完成。

## 已知限制

- 当前二维码只在机构局域网可达；离开该网络、地址变化或访客网络隔离都会导致手机无法访问。
- 单管理员配置，没有角色、审计日志和忘记密码页面；改密由服务器脚本完成。
- 固定二维码一旦实际分发，其保护引用不能由普通管理员删除，本阶段也不提供删除资料功能。
- PDF 仅支持单个二维码、单页和四角预设位置，不支持自由拖拽或批处理。
- 预览只渲染目标页；超大或复杂 PDF 的处理速度仍受 512 MiB 限制。
- 当前不是公网生产部署，没有 HTTPS、反向代理、自动备份恢复和监控告警。
