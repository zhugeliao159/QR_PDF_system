## 1. 当前结论

第二阶段已经完成并部署，当前系统可以完成以下闭环：

1. 上传一个解析文件或答案 PDF。
2. 为文件创建一个随机、唯一、不可预测的永久 `qr_id`。
3. 生成内容为 `{PUBLIC_BASE_URL}/r/{qr_id}` 的二维码 PNG。
4. 把二维码写入练习册 PDF 的指定页和四角预设位置。
5. 下载写入二维码后的 PDF。
6. 通过永久入口 `/r/{qr_id}` 访问当前绑定文件。
7. 替换绑定文件，同时保持 `qr_id`、二维码内容和入口地址不变。
8. 保留最近 5 个文件版本，支持查询和回滚。
9. 容器重启或完整 `docker compose down/up` 后保留数据。

当前状态为内部可用原型，不是公网生产系统。QuickDrop 和 PDF Worker 都在运行，并且处于 `healthy` 状态。

## 2. 当前架构和地址

| 组件 | 用途 | 远端监听地址 |
| --- | --- | --- |
| QuickDrop | 独立文件服务，当前未与二维码业务数据库耦合 | `127.0.0.1:18080` |
| PDF Worker | 永久绑定、版本、二维码和 PDF 处理 | `127.0.0.1:18081` |
| SQLite | PDF Worker 业务数据库 | `/data/db/app.db` |
| LocalStorageBackend | 绑定文件、源 PDF 和输出 PDF | `/data/storage` |

PDF Worker 当前资源限制为 1 CPU、512 MiB 内存和 128 PIDs，以非 root `appuser` 运行。服务只发布到服务器 `127.0.0.1`，没有直接暴露到局域网或公网。

## 3. 二维码和 PDF 的对应语义

当前数据关系是：

```text
唯一 qr_id -> 唯一 current_version_id -> 当前文件
```

同一个二维码在任何时刻只对应一个当前文件，但它不是永久锁死一个不可变化的 PDF。执行替换或回滚后，二维码仍然不变，当前文件会改变。

确认当前 PDF 时，应同时记录：

```text
qr_id + version_id + version_number + size_bytes + sha256
```

其中 SHA-256 用于证明实际下载文件与数据库登记内容逐字节一致。

用户已经实际验证以下绑定：

| 字段 | 值 |
| --- | --- |
| `qr_id` | `df826233e31c4b98891d50aa7d6d4cc0` |
| `version_id` | `3` |
| `version_number` | `1` |
| MIME | `application/pdf` |
| 文件大小 | `254738` bytes |
| SHA-256 | `bd4b3408f4986d4a41f5071c2d645328edf70882181438697be849331eb6f0fc` |

通过 `/r/{qr_id}` 下载后的文件 SHA-256 与数据库值相同，PowerShell 比较结果为 `True`。这证明该二维码当前入口返回的就是登记的 PDF。

## 5. 创建永久绑定

在 Swagger 中调用 `POST /bindings`，上传解析文件或答案 PDF。也可以使用 PowerShell：

```powershell
$binding = curl.exe -s `
  -F "file=@D:\files\answer.pdf;type=application/pdf" `
  -F "note=Unit 1 answer" `
  http://127.0.0.1:18081/bindings | ConvertFrom-Json

$qrId = $binding.qr_id
$binding | ConvertTo-Json -Depth 5
```

必须保存返回的 `qr_id`。二维码 PNG 可以这样下载：

```powershell
curl.exe -o qr.png `
  "http://127.0.0.1:18081/bindings/$qrId/qr.png"
```

永久入口是：

```text
http://127.0.0.1:18081/r/{qr_id}
```

## 6. 将二维码写入练习册 PDF

调用 `POST /pdf/jobs`：

```powershell
$job = curl.exe -s `
  -F "file=@D:\files\exercise.pdf;type=application/pdf" `
  -F "qr_id=$qrId" `
  -F "page=1" `
  -F "position=bottom-right" `
  -F "size_mm=20" `
  -F "margin_mm=10" `
  http://127.0.0.1:18081/pdf/jobs | ConvertFrom-Json

$job | ConvertTo-Json -Depth 5
```

参数规则：

- `page` 从 1 开始。
- `position` 支持 `top-left`、`top-right`、`bottom-left`、`bottom-right`。
- `size_mm` 支持 10 至 50，默认 20。
- `margin_mm` 支持 0 至 50，默认 10。

下载输出：

```powershell
curl.exe -o exercise-with-qr.pdf `
  "http://127.0.0.1:18081/pdf/jobs/$($job.job_id)/download"
```

## 7. 验证二维码当前对应的文件

查询绑定：

```powershell
$binding = Invoke-RestMethod `
  "http://127.0.0.1:18081/bindings/$qrId"

$binding | ConvertTo-Json -Depth 5
```

下载永久入口当前文件并比较 SHA-256：

```powershell
Invoke-WebRequest `
  "http://127.0.0.1:18081/r/$qrId" `
  -OutFile current.pdf

$actualHash = (Get-FileHash current.pdf -Algorithm SHA256).Hash.ToLower()
$expectedHash = $binding.current_version.sha256.ToLower()

$actualHash
$expectedHash
$actualHash -eq $expectedHash
```

结果为 `True` 时，说明下载文件与当前版本登记内容完全一致。

## 8. 替换、查询版本和回滚

替换当前文件：

```powershell
curl.exe -X PUT `
  -F "file=@D:\files\answer-v2.pdf;type=application/pdf" `
  -F "note=Unit 1 answer v2" `
  "http://127.0.0.1:18081/bindings/$qrId/file"
```

替换后 `qr_id` 和二维码内容不变，永久入口开始返回新文件。

查询最近 5 个版本：

```powershell
$versions = Invoke-RestMethod `
  "http://127.0.0.1:18081/bindings/$qrId/versions"

$versions | ConvertTo-Json -Depth 5
```

回滚到指定 `version_id`：

```powershell
$versionId = 3
Invoke-RestMethod -Method Post `
  "http://127.0.0.1:18081/bindings/$qrId/rollback/$versionId"
```

## 11. 当前已知问题和限制

1. 手机不能直接扫描当前二维码访问文件。二维码使用 `127.0.0.1`，在手机上该地址指向手机自身。
2. 当前接口没有登录和权限控制，不能直接暴露到公网。
3. 中文上传文件名出现了乱码显示，文件内容、大小和 SHA-256 不受影响，但文件名编码需要修复。
4. Swagger 中不删除示例值时，`note` 可能保存为字面值 `string`。
5. 当前没有面向普通用户的管理页面，主要通过 Swagger 或 API 操作。
6. 当前二维码对应唯一“当前版本”，并非不可变化的永久固定 PDF。
7. 尚未实现批量上传、批量写入、ZIP 导出、多二维码、任意坐标、用户系统、扫码统计、HTTPS 和自动备份恢复。

## 12. 接下来需要共同决定的事项

建议与下一位协作者优先确认以下问题：

1. 手机访问采用 Tailscale、局域网地址，还是正式域名和 HTTPS。
2. 产品需要“永远访问最新文件”的动态二维码，还是“永久锁定某个版本”的不可变二维码，或者两者都要。
3. 是否增加版本固定入口，例如 `/r/{qr_id}/versions/{version_id}`。
4. 是否在二维码旁打印短 ID、文件名、版本号或 SHA-256 短摘要，方便纸面核对。
5. 中文文件名如何规范编码、清洗和展示。
6. 哪些接口需要登录、管理员权限和审计日志。
7. 是否需要一个创建绑定、上传 PDF、下载输出和版本回滚的管理页面。
8. 正式数据的备份频率、保留周期和恢复演练要求。
9. 是否保存学生信息、扫码记录或来源信息，以及对应隐私边界。

在手机可达地址和鉴权方案确定前，不建议直接把 `18081` 端口发布到局域网或公网。
