# Stage 05C 性能报告

测试环境：远程机 Docker、Python 3.11、Pillow、1200×1697 WebP、ASCII 字体回退。命令：`python -m app.scripts.benchmark_watermark`。

| 场景 | P50 | P95 | 墙钟 | CPU | 错误 |
|---|---:|---:|---:|---:|---:|
| A4 文字单页 | 86.20 ms | — | — | — | 0 |
| 公式密集单页 | 100.94 ms | — | — | — | 0 |
| 彩色单页 | 71.91 ms | — | — | — | 0 |
| 20 页，串行 | 78.86 ms | 94.01 ms | 1589.29 ms | 1587.27 ms | 0 |
| 20 页，5 并发 | 110.75 ms | 142.86 ms | 485.25 ms | 1727.16 ms | 0 |
| 20 页，10 并发原始编码压力 | 164.27 ms | 206.89 ms | 414.04 ms | 1803.50 ms | 0 |

三类页面平均输出 83,740 字节。包含 1、5、10 并发连续批次的进程最大 RSS 增量高水位为 441,104 KiB；因此生产链路使用全局 6 并发硬门，10 个同时请求时最多 6 个进入编码，其余返回 429，避免 512 MiB 容器继续扩大峰值。常规懒加载 3 页未触发限速。

结论：单页即时生成无需磁盘缓存，延迟可接受；内存是主要约束，必须保留全局并发门。
