# Stage 4C 草稿与发布流程

更新时间：2026-07-15

## 业务状态

- `draft`：仅管理员可见，可预览、发布或放弃；学生公开入口拒绝访问。
- `published`：已经正式发布，可作为当前答案或历史答案，也可被锁定版本二维码引用。
- `withdrawn`：预留给未来的明确撤销能力；普通历史版本不会自动变为此状态。

当前学生答案只由资料的“当前已发布答案”决定。创建草稿不会改动它。

## 新建答案版本

1. 管理员在资料详情页点击“新建答案版本”。
2. 上传本地文件并填写可选版本说明。
3. 系统校验文件，创建独立文件资产和草稿版本，记录 `create_draft`。
4. 页面进入受登录保护的草稿预览。
5. 学生动态二维码继续显示原来的已发布答案。

草稿文件只能通过 `/admin/.../drafts/.../file` 读取。公开 `/content/{key}` 会拒绝未发布版本。

## 发布此版本

管理员在预览页二次确认后发布。服务在一个数据库事务中完成：

1. 校验资料、草稿归属和状态。
2. 校验文件存在、大小和 SHA-256。
3. 校验管理员打开页面时的资料状态没有过期。
4. 把资料当前答案切换到目标版本。
5. 把草稿标记为已发布并写入发布时间。
6. 记录 `publish_revision`。

任一步失败都会回滚，学生继续看到旧答案，草稿和文件保持不变。

## 重新发布历史版本

“重新发布此版本”只切换当前已发布答案，不复制文件、不创建重复版本，也不修改锁定版本二维码。成功后记录 `republish_revision`。

## 放弃草稿

只能放弃草稿，不能放弃当前答案、已发布答案或被引用的版本。系统先把孤立文件移入临时回收位置，在数据库事务成功后再删除；事务失败时恢复文件。共享文件资产仍被其他版本引用时不会删除。成功后记录 `discard_draft`，审计记录保留但不再保留已删除草稿的内部关联。

## 旧接口兼容

`PUT /bindings/{qr_id}/file` 保留，且仍要求管理员认证。内部流程改为“创建草稿后立即发布”，记录 `create_draft` 和 `legacy_immediate_publish`。中文后台不再调用该接口。
