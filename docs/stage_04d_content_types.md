# Stage 4D 内容类型

更新时间：2026-07-15

## 统一版本流程

答案版本支持两种技术目标、三种管理员内容选择：

| 管理员选择 | 版本目标 | 内容识别 |
| --- | --- | --- |
| 上传 PDF | 本地文件 | `application/pdf` |
| 上传图片 | 本地文件 | `image/png`、`image/jpeg`、`image/webp` |
| 使用外部网页 | 外部地址 | 经过校验的 HTTP(S) URL |

三种内容都使用“创建草稿 -> 预览 -> 显式发布 -> 历史重新发布”的统一流程。动态二维码指向当前已发布版本，锁定版本二维码指向确定版本。

## 图片规则

- 只支持 PNG、JPEG 和 WebP。
- 使用流式上传限制真实字节数，默认不超过 30 MiB。
- Pillow 按文件内容识别格式，不信任扩展名或客户端 Content-Type。
- 检查宽高、完整解码和像素数，默认最多 4000 万像素。
- Pillow 解压炸弹警告按错误处理。
- SVG、GIF、TIFF、BMP、HEIC、视频和音频不支持。
- 文件使用随机内部名称，保留清洗后的中文原文件名、真实 MIME、大小和 SHA-256。

图片草稿在受保护的管理员页面直接预览。发布后，学生页用自适应 `<img>` 立即显示，保持比例、不裁剪，可点击查看原图或下载。

## PDF 与缓存

PDF 行为保持不变，学生页继续尝试内嵌显示。PDF 和图片都通过不可变入口 `/content/{revision_key}` 读取，并使用：

```text
Cache-Control: public, max-age=31536000, immutable
ETag: "{sha256}"
```

动态 `/q` 页面和解析跳转保持 `no-store`。发布新内容不会改变旧不可变版本或锁定版本二维码。

## 外部网页体验

外部网页不会嵌入 iframe，也不会自动跳转。学生先看到统一中文页面、外部域名和风险提示，明确点击“打开外部解析”后，服务端重新解析当前版本并临时跳转。

不嵌入的原因包括第三方跟踪、恶意内容、登录状态、点击劫持、`X-Frame-Options`、CSP 和页面兼容性。
