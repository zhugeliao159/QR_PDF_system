# Stage 05 安全与产品承诺边界

系统能做到：学生端不提供原始文件下载，文件内容以私有基础 Preview 派生的逐页 WebP 展示；每次访问使用 HttpOnly Viewer Session，页面由服务端加入匿名 trace_code 水印；动态二维码跟随当前发布版本，固定二维码保持指定版本；限速、撤销、过期、中文脱敏错误、安全响应头、清理、备份校验和 Worker 重试均可执行和审计。

管理员指南统一使用以下表述：

- 学生端不提供原始文件下载。
- 内容以分页预览图片形式展示。
- 每次访问带有匿名追踪标识。
- 系统能够提高复制和传播成本。

系统无法保证：浏览器绝对无法保存已收到的 WebP，无法阻止截图、录屏、拍照、OCR、浏览器扩展或二次传播；水印不能确认真实个人身份；单机 SQLite 服务不是高可用系统。不得宣传“绝对无法下载”“完全禁止截图”“内容不会传播”。

原始 Viewer Token 只存在客户端 Cookie，服务端仅保存 HMAC；默认不保存完整 IP，只保存可选 HMAC 网络指纹和 User-Agent HMAC。Cookie 需在正式 HTTPS 环境启用 Secure。

外部 URL 不受本系统控制，可能提供下载、复制或跳转功能。默认 `PROTECTED_PREVIEW_EXTERNAL_URL_POLICY=disable`；如机构坚持启用，必须显式改配置、发布时二次确认、学生页提示离开本站，并且不得声称外部内容受到本系统的预览保护。

正式公网前至少需要：稳定域名和 HTTPS、反向代理与可信代理头策略、学生域名和管理域名/网络隔离、管理员 MFA 或等效强化、Cookie Secure、密钥轮换、集中日志和告警、异机加密备份、定期恢复演练、漏洞与依赖更新、容量测试、隐私合规确认、明确事件响应。Stage 05 未修改网络、防火墙、路由或 Tailscale，也未执行公网发布。
